Arquitectura del sistema
🎨 UI Layer
React + TypeScript — Tools page, Marketplace view, Tool detail page, instalación/progreso. El componente de cada tool (JS bundle) se carga dinámicamente en un webview sandboxed.
🗄 Store Layer
toolsStore (Zustand) — Lista de tools instaladas, estado de descarga, herramienta activa. Persiste en SQLite (tabla tools_installed).
⚙️ Tauri Commands
Rust commands — tools_fetch_registry, tools_install, tools_uninstall, tools_launch_sidecar. El sidecar de cada tool corre como proceso hijo de Tauri.
🔧 Sidecar Layer
avatar-perf-core — Binario Rust independiente. Parsea escenas Unity (YAML), cuenta meshes/materiales/PhysBones, estima VRAM. Comunica por stdin/stdout JSON. Descargado desde GitHub Releases.
☁️ Registry
tools-registry.json en GitHub — Lista de tools disponibles con metadata, versiones, URLs de descarga del JS bundle y sidecar. Cacheado localmente con TTL de 1h.

Estrategia del viewport 3D
🎬 Unity Headless Principal

El sidecar detecta la versión de Unity del proyecto, lanza Unity.exe -batchmode -executeMethod con un script C# mínimo que posiciona el avatar en T-pose, hace screenshot y cierra. Resultado: imagen PNG fiel con todos los shaders Unity. ~15–30s.

🔧 GLTF + Three.js Fallback

Si Unity no está disponible o falla, el sidecar extrae los meshes FBX/OBJ desde los Assets del proyecto y las texturas PNG/JPG, los convierte a GLTF con gltf-rs o fbx2gltf, y Three.js los renderiza con rotación en tiempo real.
