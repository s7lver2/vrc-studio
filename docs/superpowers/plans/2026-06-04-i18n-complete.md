# i18n Complete — Full Translation EN / ES / DE

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Every user-visible string in VRC Studio passes through the i18n system and has correct EN / ES / DE translations.

**Architecture:** All translations live in a single file `src/i18n/index.ts` with three locale objects (`en`, `es`, `de`). The `useT()` hook returns a typed `t(key, vars?)` function. Every component that shows text to the user must call `useT()` and replace hardcoded strings with `t("key")`. New keys must be added to all three locales simultaneously.

**Tech Stack:** TypeScript, React, custom i18n (`src/i18n/index.ts`) — no external library.

---

## How the i18n system works

```ts
// src/i18n/index.ts structure:
const translations = {
  en: { nav_projects: "Projects", ... },
  es: { nav_projects: "Proyectos", ... },
  de: { nav_projects: "Projekte", ... },
} as const;

export type TranslationKey = keyof typeof translations.en;

// Usage in component:
import { useT } from "@/i18n";
const t = useT();
t("nav_projects")           // → "Projects" / "Proyectos" / "Projekte"
t("folders_items_count", { count: 5 }) // → "5 items" (uses {count} in string value)
```

**Key naming convention:** `section_subsection_descriptor` in snake_case, e.g. `import_tags_label`, `settings_tab_connections`.

**Adding new keys:** append to `en` object, then add same key to `es` and `de`. TypeScript will error if ES/DE are missing a key from EN since the type is derived from `en`.

**Important:** The `de:` section ends just before `} as const;`. The `es:` section is between `en:` and `de:`.

- [ ] **Step 3: Update `useTour.ts` to use i18n**

The `STEPS` array currently has hardcoded strings. Since hooks can call other hooks, but `useTour` is a hook itself, it CAN call `useT()`. However, `STEPS` is defined at module level (outside the hook) which means it can't be reactive to locale changes.

**Solution:** Move `STEPS` inside the `useTour()` function body so it's recreated on each render with the current locale. Import `useT` from `@/i18n`.

```ts
// src/hooks/useTour.ts
import { useState, useCallback } from "react";
import { useT } from "@/i18n";

const TOUR_DONE_KEY = "onboarding_tour_done";

export interface TourStep {
  targetId: string;
  title: string;
  description: string;
}

export function useTour() {
  const t = useT();
  const [tourVisible, setTourVisible] = useState(false);
  const [step, setStep] = useState(0);

  const STEPS: TourStep[] = [
    { targetId: "nav-projects",  title: t("tour_step_projects_title"),  description: t("tour_step_projects_desc")  },
    { targetId: "nav-packages",  title: t("tour_step_packages_title"),  description: t("tour_step_packages_desc")  },
    { targetId: "nav-inventory", title: t("tour_step_inventory_title"), description: t("tour_step_inventory_desc") },
    { targetId: "nav-shop",      title: t("tour_step_shop_title"),      description: t("tour_step_shop_desc")      },
    { targetId: "nav-tracker",   title: t("tour_step_tracker_title"),   description: t("tour_step_tracker_desc")   },
    { targetId: "nav-settings",  title: t("tour_step_settings_title"),  description: t("tour_step_settings_desc")  },
  ];

  const startTour = useCallback(() => {
    if (localStorage.getItem(TOUR_DONE_KEY) === "true") return;
    setStep(0);
    setTourVisible(true);
  }, []);

  const complete = useCallback(() => {
    localStorage.setItem(TOUR_DONE_KEY, "true");
    setTourVisible(false);
  }, []);

  const advance = useCallback(() => {
    setStep((prev) => prev + 1);
  }, []);

  const skip = useCallback(() => {
    complete();
  }, [complete]);

  return {
    tourVisible,
    step,
    currentStep: STEPS[step] ?? null,
    totalSteps: STEPS.length,
    startTour,
    advance,
    skip,
    complete,
  };
}
```
